# 功能需求

## 功能模块概览

| 模块 | 名称 | 优先级 | 说明 |
|------|------|--------|------|
| M1 | 台账管理 | P0 | 订单的生命周期管理（预约→开台→付款→闭台） |
| M2 | 包间管理 | P0 | 包间状态可视化，预约时校验包间可用性 |
| M3 | 支付管理 | P0 | 记录支付信息，校验美团验券 |
| M4 | 报表导出 | P1 | 一键生成 Excel 财报（后续迭代） |

## 模块一：台账管理

### 订单生命周期

```
                    ┌─ 有预约（定金50元）→ 开台 ─┐
创建预约（定金50元）─┤                            ├─→ 付款 → 闭台 → 订单结束
                    └─ 爽约（未到）→ 废订单 ─────┘

直接到店（无预约）→ 开台 → 付款 → 闭台 → 订单结束
```

- **预约非必选**：顾客可直接到店开台，无需预约。
- **预约规则**：预约需支付定金 **50 元**，用于包间预留。定金在结算时退回。

### 订单字段

| 字段 | 说明 |
|------|------|
| 预约时间 | 顾客预约的到店时间 |
| 开台时间 | 顾客实际到店、开始使用包间的时间 |
| 闭台时间 | 顾客离开、包间结束使用的时间 |
| 人数 | 到店人数 |
| 包间 | 选择的包间 |
| 定金 | 预约定金，默认 50 元，允许店主调整（无预约则为 0） |
| 实付款 | 顾客实际支付的金额 |
| 退款 | 退款金额（如有） |
| 支付方式 | 是否美团验券 |
| 记录时间 | 订单创建/最后修改的时间戳 |
| 订单状态 | 预约中 / 进行中 / 已结束 / 废订单（爽约） |
| 过夜人数 | 过夜顾客人数（0 表示无过夜） |
| 过夜费 | 过夜费总额（过夜人数 × 20 元） |

### 订单修改

- 订单创建后支持修改（如换包间、改人数、改时间等）。
- 预约时间精确到几点几分，但可不填满（允许不确定的时间）。

### 录入效率优化

> 针对以下痛点设计高效录入方案：
> - **时间**：预约时间、开台时间、闭台时间 —— 尽量自动化填充（如点击"开台"自动记录当前时间）
> - **人数**：快速选择或输入
> - **包间选择**：可视化选择，自动校验时段可用性
> - **价格计算**：根据包间类型和时长自动计算费用

### 包间状态

| 状态 | 说明 |
|------|------|
| 空闲 | 未被预约、未被使用，可选 |
| 预约 | 已被预约但尚未开台，不可选 |

### 可视化要求

- 主界面需直观展示所有包间的实时状态（如不同颜色标识）
- 预约时需校验目标包间在预约时段内是否可用

## 模块三：支付管理

- 定金：默认 50 元，用于预约包间预留，允许店主调整。结算时定金退回。无预约订单无定金。
- 实付款：按实际收款金额记录。需校验最终支付是否为 **美团验券**（仅标记是/否）。
- 退款：记录退款金额（定金退回也算退款）。
- 其他收费项目：作为可扩展项（如饮料、零食等）。

### 定价规则

| 4号中包 | 298 元 | 一口价 |
