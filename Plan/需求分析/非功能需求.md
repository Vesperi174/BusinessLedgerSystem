# 非功能需求

## 性能需求

- 响应时间：系统启动 < 3 秒，页面操作响应 < 200ms。
- 并发用户数：1 人（单机桌面应用）。
- 吞吐量：无特殊要求。

## 平台与环境

- **操作系统**：Windows 10（后续兼容 Windows 11）。
- **部署方式**：安装包内集成 Java 运行环境、嵌入式数据库，开箱即用，无需用户手动安装 Java / Maven / 数据库。
- **数据库**：轻量级嵌入式数据库（H2 或 SQLite），数据库文件存储在应用目录下。

## 界面需求

- **风格**：简约美观，不花哨也不简陋，UI 组件应有高级感，视觉舒适。
- **包间状态可视化**：主界面需直观展示所有包间实时状态（空闲 / 预约 / 使用），以不同颜色区分。
- **操作便捷**：尽量减少手动输入，多用点击选择、自动填充等交互方式。

## 安全需求

- 身份认证：无需登录/认证（单人使用）。
- 数据安全：支持数据备份/恢复，防止数据丢失。
- 审计日志：记录订单操作日志，支持定长清理（保留最近 N 条）。

## 可用性需求

- 系统可用性：桌面应用，无服务端，可用性取决于本机。
- 容灾备份：支持手动备份数据库文件到指定路径。

## 可维护性需求

- 代码规范：遵循 Java 编码规范，使用清晰的包结构和命名。
- 设计模式：遵循经典设计模式，向设计目标看齐，但避免过度设计。
- 文档要求：代码注释清晰，核心逻辑有说明。
- 日志与监控：操作日志记录用户对订单的增删改操作，定长清理。
- 版本迭代：系统架构支持后续功能扩展（如报表导出、其他收费项目等）。
