# 业务需求概述

## 项目背景

- **用户身份**：桌游店老板，系统唯一使用者。
- **现状**：目前使用 Excel 手工记账，记录预约时间、开台/闭台时间、支付方式、定金、退款、实付款、订单状态等。
- **痛点**：
  - Excel 手工录入效率低，容易出错。
  - 包间状态（是否可选、是否正在使用）无法实时掌握。
  - 预约未到（爽约）情况需要手动标记，容易遗漏。
  - 缺乏一键生成 Excel 财报的能力。

## 项目目标

- **单人使用**：系统仅服务于店主一人，无需多用户、多角色权限体系。
- **提高录入效率**：尽可能简化录入操作，减少重复输入，提升记账速度。
- **轻量化部署**：工作电脑无 Java / Maven / 数据库等环境，要求安装包内集成所有依赖，开箱即用。
- **支持版本迭代**：系统架构支持后续功能扩展与升级。

## 业务范围

| 范围 | 内容 |
|------|------|
| **包含** | 台账记录（预约、开台、收付款、退款、订单状态）、包间管理（包间状态查看）、一键生成 Excel 财报 |
| **不包含** | 外部系统对接（ERP、银行等）、多用户/多角色权限、审批流程、财务核算（仅做台账记录与导出） |

- **数据库需求**：轻量级嵌入式数据库，无需单独安装数据库服务。
- **部署形态**：桌面端单机应用，安装包内集成运行环境。

## 核心业务流程

```
顾客到店
    │
    ├── 有预约（已付定金50元）→ 开台（记录人数）→ 付款 → 闭台 → 订单结束
    │
    ├── 无预约（直接到店）→ 开台（记录人数）→ 付款 → 闭台 → 订单结束
    │
    └── 预约未到（爽约）→ 订单标记为废订单（仍需记录）

创建预约 → 支付定金 50 元（包间预留），订单状态为"预约中"
```

**订单生命周期**：

- **有预约**：创建预约（定金50元）→ 开台 → 付款 → 闭台 → 结束
- **无预约**：直接开台 → 付款 → 闭台 → 结束
- **爽约**：预约 → 未到 → 废订单
